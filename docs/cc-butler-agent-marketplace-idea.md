# Shared agent-definition marketplace — future option, not scoped

## What it is

A public, marketplace-style distribution mechanism for cc-butler sub-agent
definitions (starting with `haiku-summarizer`, see
`haiku-summarization-delegation` in `governance/`): an Emacs-side `cc-butler`
installer plus a shared public plugin repo that individual worker repos pull
the agent/summarizer definitions from, so every workspace in the fleet gets
the same definitions without hand-copying files.

## Why it isn't built now

The local version — one `.claude/agents/*.md` file generated by
`cc-butler--haiku-summarizer-agent-md` and scaffolded into the butler and
steward homes by `cc-butler--ensure-butler-home` / `cc-butler--ensure-steward-home`
— just shipped and hasn't run long enough to prove itself. A shared
marketplace mechanism is a harder-to-reverse infrastructure decision (a public
repo, a pull/update mechanism, a versioning story for definitions that drift
across consumers) than a single generated file two homes already have. Building
it before the local version is proven risks locking in a distribution shape
around requirements we don't have real signal on yet.

## Recommendation

Let the local, generated-into-butler-and-steward version run for a few days.
If it holds up (workers actually delegate summarization reads, haiku is
adequate for the extraction quality needed, no drift/versioning pain shows up
across the two homes), then revisit marketplace-ization — likely starting with
just the worker-repo rollout (option 2 from the exploration that led here: hook
into `cc-butler-scaffold-functions`, the same abnormal hook the session-cleaner
already uses for the statusLine installer) before reaching for a public repo.

## Status

Recorded idea, not scoped. Do not start building this without checking back in
first — this doc exists so the idea isn't lost, not as a plan to execute.
